## 🎬 採收與匯入組 (Ingest)

### 1. `import_fb_scraper.py` (FB 採收與智慧分流 - [新版自動化])
本腳本已升級為類別化架構，支援 Cookie 儲存與全自動捲動。建議直接透過 Streamlit 介面操作。

*   **檔案位置**: `core/import_fb_scraper.py`
*   **網頁對應**: `pages/5_📱_FB採收.py`
*   **核心功能**: 
    *   **Cookie 儲存**: 初次登入後儲存，下次採收免動手。
    *   **全自動捲動**: 設定次數後，系統自動翻閱專頁採收。
    *   **智慧分流**: 關鍵字 > 50 且包含核心字眼存入精華區。

#### 🚀 使用步驟 (Streamlit 推薦)
1. 啟動 `Home.py`。
2. 切換至「📱 FB 採收」頁面。
3. 初次使用：開啟瀏覽器登入 -> 點擊「儲存 Cookie」。
4. 開始採收：設定目標 ID 與次數 -> 點擊「開始自動採收」。

#### 📥 輸出檔案
*   `fb_final_posts.txt`: 判定為精華的文章（建議直接用於機器人大腦）。
*   `fb_rejected_posts.txt`: 判定為廢文或短文（建議人工快速掃一遍，確保沒遺漏）。

#### ⚙️ 常用參數
*   `MIN_LENGTH`: 最小字數門檻（預設 50）。
*   `XIAO_KEYWORDS`: 判定為精華的關鍵字清單。

## 🏭 內容生產分流說明 (Production Lines)

為了滿足不同用途，系統將 AI 分為兩條獨立的「生產線」：

| 特性 | 小綠專案 (精品加工) | Line OA Bot (知識備份) |
| :--- | :--- | :--- |
| **主要用途** | 製作深度分析、科學解說文案 | 建立可檢索的全文知識庫 |
| **AI 處理** | **精品加工**：加入比喻、理論背景 | **全文整理**：僅校對錯字、移除雜訊 |
| **指令模板** | `[小綠精品版] SoR_SOP_v2.0` | `[LineOA 全文版] Knowledge_Backup` |
| **溫度 (Temp)** | **0.7 (創意暖色)** | **0.2 (精確冷色)** |

---

## 🎛️ AI 技術參數註記：溫度 (Temperature)

在系統設定或程式碼中，您可以微調 AI 的「脾氣」：

*   **0.2 (低溫 / 穩定)**：
    *   **感覺**：精確、死板、實事求是。
    *   **適用場景**：**Line OA Bot**。確保機器人只講資料庫裡有的話，不自行加戲編故事。
*   **0.7 (中溫 / 創意平衡)**：
    *   **感覺**：活潑、會舉一反三、有畫面感。
    *   **適用場景**：**小綠專案精品文案**。讓 AI 能根據理論延伸出精彩的比喻。

## 🤖 Line OA Bot 核心指令書 (System Prompt)

目前 Line OA 機器人 (`app.py`) 背後運行的高級指令如下，這定義了它的「蕭博士數位分身」人格：

1.  **身份帶入**：自稱「我」，身分是致力於 SoR 的語言專家。語氣專業、自信、溫厚。
2.  **同理心**：針對家長焦慮要給予「我懂你的擔心」這類的同理心互動。
3.  **實事求是**：資料庫沒答案時，嚴禁編造。要溫馨建議：「這題我還沒拍影片講過，我先記在小本子裡...」。
4.  **語音優化**：字數限制 80 字，確保語音播放長度適中。

## 🛠️ 指令管理 (`prompts.json`)

所有指令模板都存放在 `core/prompts.json`。您可以透過修改此檔案來調整生產線的「加工品質」。

*   **檔案位置**: `core/content_refinery.py`
*   **用法**: `from core.content_refinery import refine_transcript`
    *   輸入：`.txt` 逐字稿。
    *   輸出：產出包含「標準回答」與「10 組 Q&A」的知識庫檔案。
*   **注意**: 需設定 `OPENAI_API_KEY` 環境變數。
