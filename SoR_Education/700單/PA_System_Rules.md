# 臺灣雙語專利音標 (PA System) 核心視覺邏輯與口訣指南

這份文件是為開發團隊與 AI 助理建立的「永久記憶庫」。它解析了專利音標（Patent Phonetics, PA）的底層圖形生成邏輯。在未來的程式開發（如自動分鏡產生、桌遊卡牌生成、文字轉語音對齊）中，必須嚴格遵守此文件的視覺與發音對應關係。

---

## 🔵 藍色陣營：子音系統 (Consonants)

子音系統透過「形狀」、「填滿與否」以及「附加符號」來精確定義發音的物理狀態。

### 1. 聲帶震動對立 (Voicing)
這套系統解決了學員死背「有聲/無聲」的痛點，直接將發音能量視覺化：
*   **藍滿筐（實心圖形）＝ 無聲子音 (Unvoiced)**
    *   **定義**：只有氣流爆破或摩擦，聲帶不震動。
    *   **例子**：ID 17 (t), ID 6 (p), ID 22 (k), ID 38 (s), ID 4 (f)。
    *   **視覺含義**：實心代表能量（氣流）集中且純粹的噴發。
*   **藍框框（空心圖形）＝ 有聲子音 (Voiced)**
    *   **定義**：除了氣流，還伴隨明顯的聲帶震動。
    *   **例子**：ID 16 (d), ID 1 (b), ID 24 (g), ID 39 (z), ID 41 (v)。
    *   **視覺含義**：外框留空代表喉嚨裡的共鳴或震動空間。

### 2. 空間位置與氣音規則 (Position & Aspiration)
同一個發音在單字的「字首」與「字尾」有不同的視覺呈現，這是 PA 系統**最關鍵的特色**之一：
*   **母音前 (字首 / Bullets)**：單純的主體符號，代表發音要飽滿送出。
    *   例如：`b` 在 **b**ed ➔ 藍框框 反ㄅ (ID 1)。
*   **母音後 (字尾 / Checkpoints)**：圖形的右上角會出現**「兩撇（小引號）」**。
    *   **規則名稱**：字尾氣音 / 輕音規則。
    *   例如：`b` 在 we**b** ➔ 藍框框 反ㄅ + 右上兩撇 (ID 2)。
    *   **教學口訣**：看到兩撇就是「煞車指令」，提醒大腦只送短促氣音，不拖泥帶水（不可唸成 web-ber）。

### 3. 微調與特殊變體 (Modifiers)
*   **彈舌音 (Flap T/D)**：用小注音「ㄉ」或小引號修飾 (如 city)。快速帶過不爆破。
*   **摩擦音 (Affrication)**：英文字母 `t` / `d` 加上類似「ㄔ/ㄓ」的小型符號 (如 tree, dream)。
*   **吐舌音 (th 組)**：
    *   無聲 `[θ]` (ID 40)：由發音相近的 `s` (ID 38) 變形而來，代表「伸出舌頭發 ㄙ」。
    *   有聲 `[ð]` (ID 41)：由發音相近的 `z` (ID 39) 變形而來，代表「伸出舌頭發 震動的 ㄙ」。
*   **鼻音**：波浪號 `~` (如 Mom 的字尾 m)。
*   **假子音**：兩條斜線 `//` (對應 `[dʒ]`, 如 job)。

## 🔴 紅色陣營：母音系統 (Vowels)

母音系統的設計理念是「視覺合成（Visual Synthesis）」與「母語遷移（Transfer from Bopomofo）」。

### 1. 能量主次 (Energy Levels)
*   **紅滿筐（實心圖形）**：代表英語中最核心、能量最強的母音（如短母音的 æ, ɪ, ɒ 以及強勢的長母音、雙母音）。在桌遊設計中，這些是高價值的「能源卡」。
*   **紅框框（空心圖形）**：代表相對弱化、或次要的母音資源（如 ʌ/ə, ɛ）。

### 2. 形狀的借用 (Bopomofo Roots)
PA 系統極大化利用了台灣學生的注音直覺：
*   `[aɪ]` (pie) ➔ 形如注音「ㄞ」。
*   `[e]` (day) ➔ 形如拉長的注音「ㄝ」(ID 48 主體)。
*   `[ɛ]` (bed) ➔ 形如標準注音「ㄝ」(ID 45)。
*   `[ɔɪ]` (coin) ➔ 結合注音「ㄊ」(短衣音 [ɪ] ID 62)。

### 3. 樂高合成方程式 (The "Small ㄝ" Family)
母音可以透過基本圖形疊加，來指示複雜的滑音或特殊音色：
*   **右上角的「小ㄝ」修飾符**：代表保留母音 `[ɛ]` 的扁平脣形或開口度。
    *   **ID 49 蝴蝶音 `[æ]`** ＝ 大丫 (ID 42 `[ɑ]`) ＋ 小ㄝ (ID 45 `[ɛ]`)。教學口訣：「嘴型像ㄚ，聲音發ㄝ」。
    *   **ID 48 長母音 `[e]`** ＝ 橫線 ＋ 小ㄝ。教學口訣：「從ㄝ滑向ㄧ」。
    *   **ID 50 雙母音 `[aʊ]`** ＝ 橫線 ＋ 小叉叉 (或特定底圖) ＋ 小ㄝ。教學口訣：「帶有ㄝ的開口度滑向ㄠ」。
*   **圓唇滑動**：
    *   **ID 64 雙母音 `[ɔɪ]`** ＝ 大叉叉 (ID 63 `[o]`) ＋ 小ㄊ (ID 62 短衣音 `[ɪ]`)。教學口訣：「先嘟嘴成 X，再滑向小 ㄊ 的聲音」。

## 💡 AI 助理開發準則 (Developer Notes)
未來的 AI 模型或腳本在需要產出專利音標的圖形陣列（`minum`）時，必須參考這個檔案與 `字典底層資料 的副本.xlsx`。
預測發音 ID 時的判定順序：
1. 定位母音核心 (紅滿/紅框)。
2. 確認子音方位 (判斷是否觸發字尾兩撇規則)。
3. 確認子音是否有聲 (選擇藍滿/藍框)。
4. 檢查特殊單字拼寫例外 (如合音 qu, 彈舌 t)。
