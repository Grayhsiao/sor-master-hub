一、 系統核心目標
----------------------------------------------------------------
將既有的 YouTube 影片資產，透過 AI 自動化轉為 LINE OA 的「銷售型文案」，
並建立一個具備「蕭博士」人設、能聽懂語音、並用仿真人聲回應的 LINE 機器人，
最終達成 24 小時自動化導購與學員互動。


二、 系統三大模組流程 (Workflow)
----------------------------------------------------------------
本系統分為「生產」、「品管」、「銷售」三個階段，邏輯如下：

[第一階段：進貨生產] -> 腳本：multi_maker.py
1. 來源：讀取指定的 YouTube 影片或播放清單。
2. 加工：下載 MP3 -> Whisper 轉逐字稿 -> GPT-4o 改寫。
3. 產出：生成符合 SOR 原理的「痛點+解方+成交」三段式文案。
4. 儲存：自動寫入 `database.txt` (資料庫)。

[第二階段：總編品管] -> 腳本：modifier.py
1. 目的：人工介入，修正 AI 生成不完美的文案。
2. 功能：
   - 讀取 `database.txt` 選擇單元。
   - 自動調閱原始錄音檔或逐字稿。
   - 接受人類口語指令 (如：語氣軟一點)，指揮 AI 重寫。
3. 更新：將修訂後的完美版本存回 `database.txt`。

[第三階段：店面銷售] -> 腳本：app.py (核心伺服器)
1. 載入：啟動時讀取 `database.txt` 作為大腦知識庫。
2. 介面：透過 LINE OA 與真實用戶互動。
3. 互動：用戶傳語音 -> 轉文字 -> 搜尋知識庫 -> 生成回覆 -> 轉語音 -> 回傳。


三、 app.py 詳細功能解說
----------------------------------------------------------------
app.py 是本系統的「大腦」與「對外窗口」，它是一個 Flask 網頁伺服器。
它不負責生產文案，只負責「應用」資料庫來服務客戶。

主要功能模組：

1. 知識庫載入 (Knowledge Base Loading)
   - 程式啟動時，會一次性讀取 `database.txt` 的內容。
   - 將這些行銷文案作為 System Prompt (系統提示詞) 的一部分。
   - 作用：讓 AI 回答時，不是瞎聊，而是根據我們準備好的「觀念」與「銷售話術」來回答。

2. 聽覺模組 (Whisper Integration)
   - 接收 LINE 傳來的用戶錄音檔 (.m4a)。
   - 透過 OpenAI Whisper 模型將語音精準轉為文字。
   - 作用：讓機器人「聽得懂」用戶在說什麼。

3. 大腦決策模組 (GPT-4o RAG)
   - 角色設定：強制 AI 扮演「蕭博士」，語氣自信、專業、溫暖。
   - 檢索增強 (RAG)：AI 會在內建的知識庫中，尋找最適合回答用戶問題的觀念。
   - 行銷置入：嚴格規定回答結尾必須加上該觀念對應的 CTA (行動呼籲)，引導購買。
   - 字數控制：限制回答在 80 字以內，適合語音聆聽的長度。

4. 聲音生成模組 (TTS - Text to Speech)
   - 使用 OpenAI TTS-1 模型 (Onyx 聲線)。
   - 將 AI 生成的文字回覆轉為 MP3 音檔。
   - 為了讓 SOR 發音正確，程式會自動將 "SOR" 替換為 "S.O.R." 以確保朗讀停頓。

5. LINE 串接模組 (Webhook Handler)
   - 驗證：確認訊息真的來自 LINE 官方 (檢查 Signature)。
   - 判斷：區分用戶傳來的是「文字」還是「語音」。
   - 回傳：計算生成音檔的長度 (Duration)，讓 LINE 播放器能正確顯示時間長度。

----------------------------------------------------------------
四、 檔案結構說明
----------------------------------------------------------------
- app.py            : 主程式 (Flask Server)
- multi_maker.py    : 文案生產工廠
- modifier.py       : 文案修改工具
- database.txt      : 核心知識庫 (所有腳本共用)
- static/           : 存放暫存的語音檔 (app.py 自動生成)
- *.mp3 / *.txt     : 原始素材 (由 maker 下載，供 modifier 參考)

新增模組：FB 貼文抓取器 (fb_grabber.py)
================================================================
角色定位：[補充原料供應商]
功能說明：
1. 來源：針對特定 FB 粉絲專頁 (如 dr.xiao.english)。
2. 機制：使用 facebook-scraper 模擬瀏覽器行為抓取貼文。
3. 加工：GPT-4o 將「閱讀型」貼文改寫為「對話型」LINE 短文案。
4. 整合：同樣匯入 database.txt，讓機器人也能用 FB 的內容來回答問題。

特別注意：
- 必須搭配 cookies.txt (瀏覽器身分證) 才能穩定運作。
- 建議定期更新 cookies.txt 以免失效。
